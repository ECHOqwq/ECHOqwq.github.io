Java学习——Java面向对象基本概念

发表于 2019-05-02 | 更新于 2019-05-27 | 分类于 Java | 评论数：

总述:面向过程(C) 面向对象(Java、C++)
什么是面向过程?
我们首先思考”怎么按步骤实现?”并按步骤对应方法一步一步完成
适合简单、不需要协作的任务
什么是面向对象?
OOA：面向对象分析
OOD：面向对象设计
OOP：面向对象编程
需要大量协作的任务
面向对象和面向过程的总结:
解决简单问题使用面向过程，解决复杂问题宏观上使用面向对象把握，微观处理上仍然是面向过程
面向对象具有三大特征：封装性、继承性和多态性，
面向过程没有继承性和多态性，并且面向过程的封装只是封装功能，而面向对象可以封装数据和功能。
一个经典的比喻
面向对象是盖浇饭、面向过程是蛋炒饭。盖浇饭的好处就是“菜”“饭”分离，从而提高了制作盖浇饭的灵活性。
饭不满意就换饭，菜不满意换菜。用软件工程的专业术语就是“可维护性”比较好，“饭” 和“菜”的耦合度比较低。

  1. 对象的理解
    对象其实就是一个内存块用来表示一堆相关联的数据
    像一个项目里面有财务人员、技术人员、运维人员一样，一个对象包含了不同类型的数据和各种方法

  2. 对象和类的概念
    类可以看做是一个模板，系统根据类的定义来创建对象

    通过抽象创建了类，某个类的对象即为某个类的实例
一个类里面有属性(成员变量)和方法

    e.g 

    public class Stu {
    //属性（成员变量）
    int id;
    String name;
    int age; 
    Computer comp; //计算机 
    void study(){  //方法
        System.out.println("我正在学习！"+comp.brand);
    }  
    //构造方法，用于创建该类的对象，无参的构造方法可以由系统自动创建
    Stu(){ //名字一定要和类名一模一样
    }
    public static void main(String[] args){
    Stu stu=new Stu(); //建立一个Stu型对象stu
    stu.id=0101010;
    Computer c1=new Computer(); //建立一个Computer型对象c1
    c1.brand="DELL"; //给c1赋值
    stu.comp=c1; //把c1的值赋给stu对象里Computer型的成员comp
    stu.study(); //可以通过.来访问对象的相关属性
    }  //main函数是程序执行的入口，可以放在这个类里面，也可以放在别的类里面
}

class Computer{  //再创建一个Computer型的类
    String brand;
}

    注: 

    要想创建类里的对象要调用main函数在里面创造对象 
类和类之间也可以相互嵌套，如上述的Computer类和Stu类
  3. Java虚拟机的内存分析
    分为栈stack、堆heap、方法区method area
    <1> 栈
    栈描述的是方法执行的内存模型。
每个方法被调用都会创建一个栈帧(存储局部变量、操作数、方法出口等)，执行完以后会被关闭删去
它属于线程私有，不能实现线程间的共享
先进后出，后进先出
是一个连续的内存空间
    <2> 堆
    堆用于存储创建(new)好的对象(数组也是对象)，所有线程共享
堆是一个不连续的内存空间，分配灵活，速度慢
    <3> 方法区/静态区
    用来存放程序中永远是不变或唯一的内容(类信息、静态变量、字符串常量等)
所有线程共享，实质上也是堆
  4. 构造方法/构造器
    用于对象的初始化
    要点
    <1> 要用new关键字调用
<2> 构造器虽然有返回值，但不能定义返回值的类型，不能再构造器里使用return返回某个值
注意:可以直接写一个不带返回值的return用于结束该方法
<3> 如果没有定义构造器，编译器会自动定义一个无参的构造函数
<4> 构造器的方法名必须和类名一致
    e.g
    class Point {
    double x, y;
    //构造方法名要和类名一致
    public Point(double _x, double _y) {
        x = _x;
        y = _y;  
    }
    public double getDistance(Point p) {
        return Math.sqrt((x - p.x) * (x - p.x) + (y - p.y) * (y - p.y));
    }
}
public class TestConstructor {
    public static void main(String[] args) {
        Point p = new Point(3.0, 4.0);
        Point origin = new Point(0.0, 0.0);
        System.out.println(p.getDistance(origin));
    }
}
  5. 方法
    方法就是一段用来完成特定功能的代码片段。
方法用于定义该类或该类的实例的行为特征和功能实现。 
方法是类和对象行为特征的抽象。
方法很类似于面向过程中的函数。
面向过程中，函数是最基本单位，整个程序由一个个函数调用组成。
面向对象中，整个程序的基本单位是类，方法是从属于类和对象的。
    方法的调用方式：
    对象名.方法名(实参列表)
    方法的详细说明
    <1> 形式参数：在方法声明时用于接收外界传入的数据。
<2> 实参：调用方法时实际传给方法的数据。
<3> 返回值：方法在执行完毕后返还给调用它的环境的数据。
<4> 返回值类型：事先约定的返回值的数据类型，如无返回值，必须显示指定为为void。
    e.g
    public class Test20 {
     /** main方法：程序的入口 */
     public static void main(String[] args) {
          int num1 = 10;
          int num2 = 20;
          //调用求和的方法：将num1与num2的值传给add方法中的n1与n2
          // 求完和后将结果返回，用sum接收结果
          int sum = add(num1, num2);
          System.out.println("sum = " + sum);//输出：sum = 30
          //调用打印的方法：该方法没有返回值
          print();
          }
          /** 求和的方法 */
          public static int add(int n1, int n2) {
               int sum = n1 + n2;
               return sum;//使用return返回计算的结果
          }
          /** 打印的方法 */
          public static void print() {
               System.out.println("HAHA");
          }
}
  6. 方法的重载
    方法的重载是指一个类中可以定义多个方法名相同，但参数不同的方法。
    调用时，会根据不同的参数自动匹配对应的方法。
    注意：
    <1> 只有返回值不同，不构成方法的重载
    int a(String str){}  void a(String str){}
<2> 只有形参的名称不同，不构成方法的重载
    int a(String str){}  int a(String s){}
    总:
    public static int add(int n1, int n2) {
     int sum = n1 + n2;
     return sum;
}
// 方法名相同，参数个数不同，构成重载
public static int add(int n1, int n2, int n3) {
     int sum = n1 + n2 + n3;
     return sum;
}
// 方法名相同，参数类型不同，构成重载
public static double add(double n1, int n2) {
     double sum = n1 + n2;
     return sum;
}
// 方法名相同，参数顺序不同，构成重载
public static double add(int n1, double n2) {
     double sum = n1 + n2;
     return sum;
}
//编译错误：只有返回值不同，不构成方法的重载
public static double add(int n1, int n2) {
     double sum = n1 + n2;
     return sum;
}
//编译错误：只有参数名称不同，不构成方法的重载
public static int add(int n2, int n1) {
     double sum = n1 + n2;         
     return sum;
}
  7. 构造方法的重载
    e.g
    public class User {
    int id; // id
    String name; // 账户名
    String pwd; // 密码
    public User() {
    }
    public User(int id, String name) {
        super();
        this.id = id; //this.id表示的是最前面的int id的id，而后面的id是该类里的形参int id
        this.name = name;
    }
    public User(int id, String name, String pwd) {
        this.id = id;
        this.name = name;
        this.pwd = pwd;
    }
    public static void main(String[] args) {
        User u1 = new User();
        User u2 = new User(101, "小七");
        User u3 = new User(100, "fafa", "123456");     
    }
}
    this的用法 
    如果方法构造中形参名与属性名相同时，需要使用this关键字区分属性与形参
this.id 表示属性id，id表示形参id
  8. this
    对象创建的过程
    分配对象空间，并对对象成员变量进行初始化
执行属性值的显示初始化
执行构造方法
返回对象的地址给相关的变量
    this的本质
    创建好的对象的地址
    this的用法
    <1>普通方法中，this总是指向调用该方法的对象
构造方法中，this总是指向正要初始化的对象
<2> 使用this关键字调用重载的构造方法，避免相同的初始化代码
但只能在构造方法中用，并且必须位于构造方法的第一句
<3> this不能用于static方法中，因为static在方法区，里面没有对象
    e.g
    public class TestThis {
    int a, b, c;
    TestThis() {
        System.out.println("正要初始化一个Hello对象");
    }
    TestThis(int a, int b) {
        // TestThis(); //这样是无法调用构造方法的！
        this(); // 调用无参的构造方法，并且必须位于第一行！
        // 这样就区分了成员变量和局部变量
        this.a = a;
    }
    TestThis(int a, int b, int c) {
        this(a, b); // 调用带参的构造方法必须位于第一行
        this.c = c;
        }
    void sing() {
    }
    void eat() {
        this.sing(); // 调用本类中的sing，也可不用this
    }
    public static void main(String[] args) {
        TestThis hi = new TestThis(2, 3);
        hi.eat();
    }
}

  9. static关键字 

        static修饰的成员变量和方法从属于类
    普通变量和方法从属于对象  
注意:静态方法里不能调用非静态方法，但非静态方法里可以调用静态方法
    public class User{
        int id;
        String name;
        public User(int id,String name){
            this.id=id;
            this.name=name;
        }
        public void login(){
            printCompany(); //可以在非静态方法里调用静态方法
        }
        public static void printCompany(){
            login(); //调用非静态方法，编译会报错
        }
        public static void main(String[] args){
            User u=new User(1111,fafa);
            User.printCompany();
        }
    }

因为它们一个在堆里一个在静态区里，堆可以找到静态区，静态区找不到堆

    构造方法用于对象的初始化
静态初始化块用于类的初始化
  11. 参数传值机制
    方法中所有参数都是值传递，也就是传递的是值的副本
    个人理解:
    把要传输的对象的值和地址都做出了一个副本给接收对象
之后改变副本里对象的值，意味着原件该值也会被改变
    这里抛出一个小问题:Java里的变量有没有地址?
    查了好多地方都找不到答案啊唔，如果有大佬看到跪求解答!
    e.g
    public class User{
    int id;        //id
    String name;   //账户名
    public User(int id, String name) {
        this.id = id;
        this.name = name;
    }
    public void testParameterTransfer01(User u){ //拷贝到该副本
        u.name="fa"; //该副本改变成员name的值，原件也会改变，因为该副本的地址是原件的地址
    }
    public   void   testParameterTransfer02(User u){ //拷贝到该副本
        u  =  new  User(100,"fafafa");} //新建了一个对象，把该对象的内容拷贝给了该副本，包括地址，这时和最初新建的u1已经没有关系了
    public static void main(String[] args) {
        User u1  =  new User(100, "FAFA"); //初始化name为FAFA
        u1.testParameterTransfer01(u1); 
        System.out.println(u1.name); //fa
        u1.testParameterTransfer02(u1);
        System.out.println(u1.name); //fa
    }
}
  12. 包

注意包里不能有重名的类
    包的处理(package)
    通常是类的第一句非注释性语句
例: package cn.ECHOqwq(倒域名).user(内容)
包名:域名倒着写
在硬盘其实就是文件夹
    JDK中提供了很多包
    注意里面有一个java.lang是核心包，String、Math等等都在这个包里
    使用该包不需要导入
  13. import
    情境导入说明import的作用:
    在一个Java项目下，用户新建了两个包A和B，A包里有a类，B包里有b类，这时用户想在A包的a类里调用b类
b b1 = new b();
编译会报错，因为A包里不存在b类，那么应该怎么办呢
B包名.b b1 = new B包名.b();
这样告诉了编译器b类所在的位置，就可以找到b类并使用它
但这样做在实操中太繁琐了，因此需要import来导入类
import B包名.b;
b b1 = new b();
    那么问题又来了，如果用户要使用B包里的好多类呢？
    import B包名.*; //但会降低编译速度，不会降低运行速度
    如果导入的包有同名的类怎么办？
    B包名.b b1 = new B包名.b(); //用这种办法可读性更强，也不容易出错
    静态导入
    未导入时调用PI
System.out.println(Math.PI);
导入后调用PI
import static java.lang.Math.*; //也可Math.PI
System.out.println(PI);
