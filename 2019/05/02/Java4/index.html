Java学习——面向对象三大特征

发表于 2019-05-02 | 分类于 Java | 评论数：

面向对象三大特征:继承、封装、多态

  1. 继承
    子类继承了父类，是父类的扩展
    e.g
    public class Extends{
    public static void main(String[] args){
        Student stu = new Student("fafa","计算机专业");
        stu.rest();
    }
}
class Person{
    String name;
    int height;
    public void rest(){
        System.out.println("休息");
    }
}
class Student extends Person{  //继承了Person的name、height、rest()
    String major;
    public void study(){
        System.out.println("学习");
    }
    public Student(String name,String major){
        this.name=name;
        this.major=major;
    }
}
    注意:
    <1> 父类也称作超类、基类、派生类
<2> Java中的类只有单继承，多继承难以维护
<3> Java中接口有多继承
<4>子类可以继承父类，得到父类的全部属性和方法(除了父类的构造方法)
   但不能直接访问
<5>如果定义一个类时，没有调用extends，它的默认父类是java.lang.Object
    小技巧:
    选中类后按ctrl+t可以查看它的层次
    instanceof运算符
    instanceof是二元运算符，左边是对象，右边是类
当对象是右边类或者子类所创建的对象时返回true，否则返回false
  2. 方法的重写override
    子类通过重写父类的方法，可以用自身的行为替换父类的行为
    方法的重写是实现多态的必要条件
    三要点
    <1> 方法名、形参列表要相同
<2> 返回值类型和声明异常类型，子类小于父类
<3> 访问权限，子类大于等于父类(涉及到封装)
  3. 介绍Object类
    是所有Java类的根基类
  4. Object类里的toString方法的改写
    toString源码
    public String toString(){
    return getClass().getName()+"@"+Integer.toHexString(hashCode());
}
    e.g
    class Person {
    String name;
    int age;
    @Override  //这是改写的注释
    public String toString() { //改写了Object类里的toString方法
        return name+",年龄："+age;
    }
}
public class Test {
    public static void main(String[] args) {
        Person p=new Person();
        p.age=18;
        p.name="fafa";
        System.out.println("info:"+p); //因为在p中改写toString方法，所以输出的是名字年龄
        Test t = new Test(); //而Test类中并没有改变toString方法，所以输出时它默认的设置
        System.out.println(t);
    }
}
  5. Object里的equals方法的改写
    “==”
    代表比较双方是否相同
如果是基本类型则表示值相等
如果是引用类型则表示地址相等
    源代码(提供定义“对象内容相等”的逻辑)
    public boolean equals(Object obj){
    return (this == obj);
}
默认就是比较两个对象的hashcode，是同一个对象的引用时返回 true 否则返回 false
    e.g
    public class Equals{
    public static void main(String[] args){
        Object obj;
        String str;
        User u1 = new User(100,"fafa",123);
        User u2 = new User(1000,"haha",123456);
        System.out.println(u1.equals(u2)); //比较u1和u2的地址，如果这不是用户想比较的内容，就进行改写
    }
}
class User{
    int id;
    String name;
    String pwd;
    public User(int id,String name,String pwd){
        this.id = id;
        this.name = name;
        this.pwd = pwd;
    }
    @Override
    public boolean equals(Object obj){
        if (this == obj)
          return true;
        if (obj ==null)
          return false;
        if (getClass()!=obj.getClass()) //这里getClass是取类的类型
          return false;
        User other = (User)obj; //这一步强制类型转化目前还很懵
        if (id!=other.id)
          return false;
        return true;
    }
}
    e.g
    public class TestEquals { 
    public static void main(String[] args) {
        Person p1 = new Person(1233,"fafa");
        Person p2 = new Person(1233,"gaga");
        System.out.println(p1==p2);     //false，不是同一个对象
        System.out.println(p1.equals(p2));  //true，id相同则认为两个对象内容相同
        String s1 = new String("fafafa");
        String s2 = new String("fafafa");
        System.out.println(s1==s2);         //false, 两个字符串不是同一个对象
        System.out.println(s1.equals(s2));  //true,  两个字符串内容相同
    }
}
class Person {
    int id;
    String name;
    public Person(int id,String name) {
        this.id=id;
        this.name=name;
    }
    public boolean equals(Object obj) {
        if(obj == null){
            return false;
        }
        else {
            if(obj instanceof Person) {
                Person c = (Person)obj;
                if(c.id==this.id) {
                    return true;
                }
            }
        }
        return false;
    }
}
