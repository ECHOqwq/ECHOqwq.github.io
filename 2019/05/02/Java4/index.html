Java学习——面向对象三大特征

发表于 2019-05-02 | 更新于 2019-05-27 | 分类于 Java | 评论数：

面向对象三大特征:继承、封装、多态

  1. 继承
    子类继承了父类，是父类的扩展
    e.g
    public class Extends{
    public static void main(String[] args){
        Student stu = new Student("fafa","计算机专业");
        stu.rest();
    }
}
class Person{
    String name;
    int height;
    public void rest(){
        System.out.println("休息");
    }
}
class Student extends Person{  //继承了Person的name、height、rest()
    String major;
    public void study(){
        System.out.println("学习");
    }
    public Student(String name,String major){
        this.name=name;
        this.major=major;
    }
}
    注意:
    <1> 父类也称作超类、基类、派生类
<2> Java中的类只有单继承，多继承难以维护
<3> Java中接口有多继承
<4>子类可以继承父类，得到父类的全部属性和方法(除了父类的构造方法)
   但不能直接访问
<5>如果定义一个类时，没有调用extends，它的默认父类是java.lang.Object
    小技巧:
    选中类后按ctrl+t可以查看它的层次
    instanceof运算符
    instanceof是二元运算符，左边是对象，右边是类
当对象是右边类或者子类所创建的对象时返回true，否则返回false
  2. 方法的重写override
    子类通过重写父类的方法，可以用自身的行为替换父类的行为
    方法的重写是实现多态的必要条件
    三要点
    <1> 方法名、形参列表要相同
<2> 返回值类型和声明异常类型，子类小于父类
<3> 访问权限，子类大于等于父类(涉及到封装)
  3. 介绍Object类
    是所有Java类的根基类
  4. Object类里的toString方法的改写
    toString源码
    public String toString(){
    return getClass().getName()+"@"+Integer.toHexString(hashCode());
}
    e.g
    class Person {
    String name;
    int age;
    @Override  //这是改写的注释
    public String toString() { //改写了Object类里的toString方法
        return name+",年龄："+age;
    }
}
public class Test {
    public static void main(String[] args) {
        Person p=new Person();
        p.age=18;
        p.name="fafa";
        System.out.println("info:"+p); //因为在p中改写toString方法，所以输出的是名字年龄
        Test t = new Test(); //而Test类中并没有改变toString方法，所以输出时它默认的设置
        System.out.println(t);
    }
}
  5. Object里的equals方法的改写
    “==”
    代表比较双方是否相同
如果是基本类型则表示值相等
如果是引用类型则表示地址相等
    源代码(提供定义“对象内容相等”的逻辑)
    public boolean equals(Object obj){
    return (this == obj);
}
默认就是比较两个对象的hashcode，是同一个对象的引用时返回 true 否则返回 false
    e.g
    public class Equals{
    public static void main(String[] args){
        Object obj;
        String str;
        User u1 = new User(100,"fafa",123);
        User u2 = new User(1000,"haha",123456);
        System.out.println(u1.equals(u2)); //比较u1和u2的地址，如果这不是用户想比较的内容，就进行改写
    }
}
class User{
    int id;
    String name;
    String pwd;
    public User(int id,String name,String pwd){
        this.id = id;
        this.name = name;
        this.pwd = pwd;
    }
    @Override
    public boolean equals(Object obj){
        if (this == obj)
          return true;
        if (obj ==null)
          return false;
        if (getClass()!=obj.getClass()) //这里getClass是取类的类型
          return false;
        User other = (User)obj; //这一步强制类型转化目前还很懵
        if (id!=other.id)
          return false;
        return true;
    }
}
    e.g
    public class TestEquals { 
    public static void main(String[] args) {
        Person p1 = new Person(1233,"fafa");
        Person p2 = new Person(1233,"gaga");
        System.out.println(p1==p2);     //false，不是同一个对象
        System.out.println(p1.equals(p2));  //true，id相同则认为两个对象内容相同
        String s1 = new String("fafafa");
        String s2 = new String("fafafa");
        System.out.println(s1==s2);         //false, 两个字符串不是同一个对象
        System.out.println(s1.equals(s2));  //true,  两个字符串内容相同
    }
}
class Person {
    int id;
    String name;
    public Person(int id,String name) {
        this.id=id;
        this.name=name;
    }
    public boolean equals(Object obj) {
        if(obj == null){
            return false;
        }
        else {
            if(obj instanceof Person) {
                Person c = (Person)obj;
                if(c.id==this.id) {
                    return true;
                }
            }
        }
        return false;
    }
}

  6. super关键字以及继承树追溯
    supper是直接父类对象的引用

    <1> 可以通过supper来访问父类中被子类覆盖的方法或属性
<2> 使用super调用普通方法，语句没有位置限制，可以在子类中随便调用
<3> 若构造方法的第一行代码没有显式的调用super/this，会默认调用super()
    含义是会自动调用父类的无参数的构造方法，这里的super()可以省略
划重点:这里说的是构造方法，不是普通方法啊!!!

    e.g

    public class TestSuper01 { 
    public static void main(String[] args) {
        new ChildClass().f();
    }
}
class FatherClass {
    public int value;
    public void f(){
        value = 100;
        System.out.println ("FatherClass.value="+value);
    }
}
class ChildClass extends FatherClass {
    public int value; //子类中再定义一个value
    /*若去掉上行，则尾句的输出结果会变成200*/
    public void f() { 
        super.f();  //调用父类对象的普通方法
        value = 200;
        System.out.println("ChildClass.value="+value);
        System.out.println(value); //200
        System.out.println(super.value); //在子类中调用父类对象的成员变量  100
    }
}

    e.g(针对上面的<3>)

    public class TestSuper02{
    public static void main(String[] args){
        System.out.println("HAHA");
        new ChildClass2();
    }
}
class FatherClass2{
    public FatherClass2(){
        super(); //可写可不写，反正默认会有这一条，调用Object类里的构造函数
        System.out.println("创建FatherClass");
    }
}
class ChildClass2 extends FatherClass2{
    public ChildClass2(){
        super(); //可写可不写，反正默认都有，调用FatherClass2类里的构造函数
        System.out.println("创建ChildClass");
    }
}

输出：HAHA
     创建FatherClass
     创建ChildClass

    从上例引出继承树追溯

    main方法先调用了ChildClass2类，进入该类的构造函数
通过默认的super()语句自动调用FatherClass2类的构造函数
再通过FatherClass2类的构造函数的默认的super()调用Object类的super()
所以会先调用Object的静态初始化块，再调用FatherClass2的静态初始化块，最后调用ChildClass2的静态初始化块

    归纳

    构造方法调用顺序：
构造方法第一句总是super()来调用父类对应的构造方法
流程就是：先向上追溯到Object，然后再依次向下执行类的初始化块和构造方法，直到当前子类为止
注：静态初始化块调用顺序，与构造方法调用顺序一样

    属性/方法查找顺序：(比如：查找变量h)

    <1> 查找当前类中有没有属性/方法h
<2> 依次上溯每个父类，查看每个父类中是否有h，直到Object
<3> 如果没找到，则出现编译错误
<4> 只要找到h变量，过程终止

  7. 封装
    定义 

    封装就是把对象的属性和操作结合为一个独立的整体，并尽可能隐藏对象的内部实现细节
好比我们运行一个网页，只能看到它的页面并对其进行操作，但不能直接看到它的复杂源代码

    追求

    我们程序设计要追求“高内聚，低耦合”
高内聚就是类的内部数据操作细节自己完成，不允许外部干涉
低耦合是仅暴露少量的方法给外部使用，尽量方便外部调用

    封装优点

    <1> 提高代码的安全性
<2> 提高代码的复用性
<3> 高内聚：封装细节，便于修改内部代码，提高可维护性
<4> 低耦合：简化外部调用，便于调用者使用，便于扩展和协作

    没有封装的问题

    程序太开放，就像出考题时如果是全开放的，会有很多人随意篡改考题，导致出现问题

    实现封装——使用访问控制符

    Java使用访问控制符来控制哪些细节需要封装，哪些细节需要暴露
    Java中4种访问控制符分别为private、default、protected、public
    它们说明了面向对象的封装性，我们利用它们让访问权限降到最低，提高安全性

    <1> private 私有，只有自己类能访问
<2> default 没有修饰符修饰，只有同一个包的类能访问
<3> protected 可以被同一个包的类以及其他包中的子类访问
<4> public 可以被该项目的所有包中的所有类访问

    在普通的开发中如何使用这四种控制符?

    <1> 对于类的属性都用private
    private int id;
<2> 有相应的get/set方法来访问public修饰的相关属性
    以提供对属性的赋值与读取操作
    注意：boolean变量的get方法是is开头
<3> 一些只用于本类的辅助性方法可以用private修饰，希望其他类调用的方法用public修饰

    e.g

    public class Person {
    // 属性一般使用private修饰
    private String name;
    private int age;
    private boolean flag;
    // 为属性提供public修饰的set/get方法
    public String getName() {
        return name;
    }
    public void setName(String name) {
        this.name = name;
    }
    public int getAge() {
        return age;
    }
    public void setAge(int age) {
        this.age = age;
    }
    public boolean isFlag() {
    // 注意：boolean类型的属性get方法是is开头的
        return flag;
    }
    public void setFlag(boolean flag) {
        this.flag = flag;
    }
}
注意:若成员过多时，一条一条打会很繁琐，可右键有一个源码，里面可以生成

    e.g

    public class Private {
    public static void main(String[] args) {
        Person p1 = new Person();
        p1.age = 11; //error!因为age被private修饰不能直接访问了
        p1.setAge(11);

  8. 多态
    定义
    同一种方法调用，由于对象不同可能会有不同的行为
    要点
    <1> 多态是方法的多态，不是属性的多态
<2> 多态的存在要有3个必要条件：继承，方法重写，父类引用指向子类对象
<3> 父类引用指向子类对象后，用该父类引用调用子类重写的方法，此时多态就出现了
    e.g
    class Animal {
    public void shout() {
        System.out.println("叫了一声！");
    }
}
class Dog extends Animal {
    public void shout() {
        System.out.println("旺旺旺！");
    }
    public void seeDoor() {
        System.out.println("看门中....");
    }
}
class Cat extends Animal {
    public void shout() {
        System.out.println("喵喵喵喵！");
    }
}
public class TestPolym {
    public static void main(String[] args) {
        Animal a1 = new Cat(); // 向上可以自动转型
        //传的具体是哪一个类就调用哪一个类的方法,大大提高了程序的可扩展性
        animalCry(a1);
        Animal a2 = new Dog();
        animalCry(a2);//a2为编译类型，Dog对象才是运行时类型
        //编写程序时，如果想调用运行时类型的方法，只能进行强制类型转换
          否则通不过编译器的检查
        Dog dog = (Dog)a2;//向下需要强制类型转换
        dog.seeDoor();
    }
    // 有了多态，只需要让增加的这个类继承Animal类就可以了。
    static void animalCry(Animal a) {
        a.shout();
    }
    /* 如果没有多态，我们这里需要写很多重载的方法
    每增加一种动物，就需要重载一种动物的喊叫方法*/
    static void animalCry(Dog d) {
        d.shout();
    }
    static void animalCry(Cat c) {
        c.shout();
    }
}
  9. 对象的转型(casting)
    向上转型
    父类引用指向子类对象，属于自动类型转换
    向下转型
    向上转型后的父类引用变量只能调用它编译类型的方法(即它自带的方法)
不能调用它运行时类型的方法(即向上转型前的子类的方法)
这时，我们就需要进行类型的强制转换，即向下转型
    对于上例把Animal强转为Dog
    Animal a2 = new Dog(); //编译器默认a2是Animal类型
a2.seeDoor(); //报错，因为Animal类型没有seeDoor这个方法
Dog a3 = (Dog)a2; //向下需要强制类型转换
a3.seeDoor();
    可以把Cat转为Dog吗
    Animal c = new Cat();
Dog c1 = (Dog) c;
c1.seeDoor();
这样编译器不会报错，但运行时会出现
java.lang.ClassCastException(类型转化错误)
在向下转型过程中，必须将引用变量转成真实的子类类型(运行时类型)
即是原来自动向上转型的那个子类的类型，不能是其他子类
否则会出现类型转换异常ClassCastException
因此不能把Cat转为Dog
    为避免上述情况，可使用instanceof来比较
    public class TestCasting3 {
    public static void main(String[] args) {
        Object obj = new String("北京尚学堂");
        if(obj instanceof String){
            String str = (String)obj;
            System.out.println(str.charAt(0));
        }else if(obj instanceof StringBuffer){
            StringBuffer str = (StringBuffer) obj;
            System.out.println(str.charAt(0));
        }
    }
}
    e.g
    public class TestCasting {
    public static void main(String[] args) {
        Object obj = new String("哈哈"); // 向上可以自动转型
        // obj.charAt(0) 无法调用，编译器认为obj是Object类型而不是String类型
        /* 编写程序时，如果想调用运行时类型的方法，只能进行强制类型转换。
           不然通不过编译器的检查。 */
        String str = (String) obj; // 向下转型
        System.out.println(str.charAt(0)); // 位于0索引位置的字符
        System.out.println(obj == str); // true.他们俩运行时是同一个对象
    }
}
  10. final关键字
    <1> 修饰变量
    被它修饰的变量不可改变，即一旦赋了初值，就不能被重新赋值
final int a =120;
    <2> 修饰方法
    该方法不可被子类重写，但可以被重载
final void study(){
}
Cannot override the final method from ……(类名)
    <3> 修饰类
    修饰的类不能被继承
final class A {}
The type ……(类名) cannot subclass the final class ……(类名)
  11. 抽象类
    定义
    包含抽象方法的类就是抽象类，通过abstrat方法定义
注意，抽象类中一定要有抽象方法，但也可以有非抽象方法
    作用
    是一种模板形式，为所有子类提供了一个通用模板，子类可以在这个模板上扩展
可以避免子类设计的随意性，使子类之间更加通用
    e.g
    abstract class Animal {
    abstract public void shout();  //抽象方法
}
class Dog extends Animal { 
    //子类必须实现父类的抽象方法，否则编译错误
    public void shout() {
        System.out.println("汪汪汪！");
    }
    public void seeDoor(){
        System.out.println("看门中....");
    }
}
//测试抽象类
public class TestAbstractClass {
    public static void main(String[] args) {
        Dog a = new Dog();
        a.shout();
        a.seeDoor();
    }
}
    要点
    <1> 有抽象方法的类只能定义成抽象类
<2> 抽象类不能实例化，即不能用new来实例化抽象类
<3> 抽象类可以包含属性、方法、构造方法，但是构造方法不能用来new实例，只能用来被子类调用
<4> 抽象类只能用来被继承
<5> 抽象方法必须被子类实现
  12. 接口
    定义
    接口就是比抽象类还抽象的抽象类
抽象类还提供某些具体实现，接口不提供任何实现，接口中所有方法都是抽象方法
从接口的实现者角度看，接口定义了可以向外部提供的服务
从接口的调用者角度看，接口定义了实现者能提供那些服务
    比喻
    好比你是一个公司的人事管理人员，你构造了一个人事管理的框架
需要每个人往框架里填他们的信息，框架是定死的不能修改的
但每个人都要可以查看框架才可以填写信息(所以不能用private)
    关系
    接口和实现类不是父子关系，是实现规则的关系
    声明格式
    访问修饰符 interface 接口名 [extends  父接口1，父接口2…]{
    常量定义；  
    方法定义；
}
<1> 其中访问修饰符只能是public或默认
<2> public修饰的接口名和类名相同
<3> 接口可以多继承
<4> 接口的属性只能是常量
<5> 接口中的方法只能是public abstract
    要点
    <1> 子类通过implements来实现接口中的规范
<2> 接口不能创建实例，但是可用于声明引用变量类型
<3> 一个类实现了接口，必须实现接口中所有的方法，并且这些方法只能是public的
    e.g
    public class TestInterface {
    public static void main(String[] args) {
        Volant volant = new Angel(); //可以把接口理解成父类(注意是理解!)
        volant.fly();
        System.out.println(Volant.FLY_HIGHT);
        Honest honest = new GoodMan();
        honest.helpOther();
    }
}
/**飞行接口*/
interface Volant { 
    int FLY_HIGHT = 100;  // 总是：public static final类型的；
    /*public abstract*/void fly();   //总是：public abstract void fly();
}
/**善良接口*/
interface Honest { 
    void helpOther();
}
/**Angle类实现飞行接口和善良接口*/
class Angel implements Volant, Honest{
    public void fly() {
        System.out.println("我是天使，飞起来啦！");
    }
    public void helpOther() {
        System.out.println("扶老奶奶过马路！");
    }
}
class GoodMan implements Honest {
    public void helpOther() {
        System.out.println("扶老奶奶过马路！");
    }  
}
class BirdMan implements Volant {
    public void fly() {
        System.out.println("我是鸟人，正在飞！");
    }
}
    接口的多继承
    接口支持多继承，子接口扩展父接口，将有父接口的一切
    e.g
    interface A {
    void testa();
}
interface B {
    void testb();
}
/**接口可以多继承：接口C继承接口A和B*/
interface C extends A, B {
    void testc();
}
public class Test implements C {
    public void testc() {
    }
    public void testa() {
    }
    public void testb() {
    }
}
