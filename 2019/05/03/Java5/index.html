Java学习——构造方法&&普通方法

发表于 2019-05-03 | 分类于 Java | 评论数：

因为自己学着学着就老弄混一些概念，所以这边做一个小小的总结

  1. 构造方法:
    <1> 方法名一定要和类名相同
<2> 在方法名的前面没有返回值类型的声明，但它可以用return;退出该方法
<3> 在方法中不能使用return语句返回一个值
<4> 在创建对象时，要调用new，如:book b1=new book();
<5> 当没有指定构造方法时，系统会自动添加无参的构造方法
<6> 构造方法的重载：方法名相同，但参数不同的多个方法，调用时会自动根据不同的参数选择相应的方法
<7> 构造方法前默认有一句super();，也正因为此构造方法不会被继承
<8> 接口不允许被实例化，所以接口没有构造方法
<9> 不能被static、final、synchronized、abstract和native修饰
    e.g (一个注意点)
    public class Text{
    public static void main(String[] args){
        Person p1 = new Person();
        p1.age = 18;
        Person p2 = new Person(19);
        System.out.println(p1);
        System.out.println(p2);
    }
}
class Person{
    int age;
    public Person(int age){
        super();
        this.age=age;
    }
    public String toString(){
        return "Person : age="+age; 
    }
}
    <1> 在System.out.println(p1)时会报错
        因为后来声明了一个带参构造函数，系统就不会自动添加无参的构造函数
    编译器就会提示不存在对应的构造函数
解决办法:再写一个构造函数
    public Person(){
    }
    <2> 重写toString的作用
    重写toString后输出p1即可输入想要的成员内容
若不改写就是系统默认的(目前还不懂那一串数字加字母有什么用)
  2. 普通方法
    <1> 无自带super();
<2> 名字不与类名相同
<3> 有返回值类型的声明
