Java学习——集合框架collection

发表于 2019-05-14 | 更新于 2019-05-27 | 分类于 Java | 评论数：
  1. 和数组的区别
    <1> 数组
    数组有长度、数据类型的限制，操作不方便
    <2> 集合
    存储元素个数不受限制、元素类型不受限制的数据存储结构
  2. 集合分类
    <1> Java中的集合使用Collection接口和Map接口表示
    <2> Collection接口下还有两个子接口，分别是List和Set
    <3> 特点
    a. Collection 接口存储一组不唯一，无序的对象
b. List接口存储一组不唯一，有序的对象，有序是指有下标
c. Set 接口存储一组唯一，无序的对象
d. Map接口存储一组键值对象，提供key到value的映射
  3. 集合框架的实现类
    <1> 在集合框架的实现类中提供了相应的操作方法
    集合中的接口及其相关类都位于java.util包中
如何向集合中添加元素
如果从集合中获取某一个元素
如何获得集合中所有的元素
如何将一个元素从集合中移除
如何判断集合中是否包含了某个元素
  4. List接口
    <1> 和数组的区别
    a. 数组长度在使用前必须确定，一旦确定不能改变
   而List集合长度可变，无需定义
b. 数组中必须存放同一类型的数据
   List集合中可以存放不同类型的数据
    <2> 常用方法
    add(Object obj)    加入元素，返回是否添加成功
clear()    清除集合中的元素
contains(Object obj)    查找集合中是否存在传入的元素get(int index)    获取指定位置的元素
isEmpty()    判断集合是否为空
remove(int index)    删除制定位置的元素，并返回该元素size()    获取集合大小
toArray()    将集合转换成一个数组
    <3> e.g
      import java.util.ArrayList;
  import java.util.List;
  class Student{
      public void study(){
          System.out.println("学生在学习");
      }
  }
  class Teacher{
      public void teach(){
          System.out.println("老师在上课");
      }
  }
  public class ArrayListDemo1 {
      public static void main(String[] args) {
          // 定义集合对象list
          List list = new ArrayList();
          // 向集合中添加不同类型的元素
          Teacher t =new Teacher();
          Student s =new Student();
          list.add("Hello");
          list.add(t);
          list.add(s);
          // 判断集合中是否包含"Hello"
          if (list.contains("Hello")) {
              System.out.println("集合中包含Hello");
          } else {
              System.out.println("集合中不包含Hello");
            }
            // 显示集合中所有的元素
          System.out.println("集合中有以下元素");
          for (int i = 0; i < list.size(); i++) {
              if(list.get(i) instanceof java.lang.String){
                  System.out.println(list.get(i));
                  }else if(list.get(i) instanceof Teacher){
                      Teacher tea =(Teacher) list.get(i);
                      tea.teach();
                  }else if(list.get(i) instanceof Student){
                      Student stu = (Student)list.get(i);
                      stu.study();
                  }
          }
          // 从集合中移除下标为0的元素
          list.remove(0);
          // 显示移除后集合中所有的元素
          System.out.println("移除后集合中还有" + list.size() +"个元素");
      }
    }
输出：
    集合中包含Hello
    集合中有以下元素
    Hello
    老师在上课
    学生在学习
    移除后集合中还有2个元素
注:
    在main方法中创建了集合对象list
    使用add方法向集合中添加了String类型的元素"Hello"，Teacher类型的元素t，Student类型的元素s
    使用contains()方法判断集合中是否包含"Hello"元素
    Size()方法用于获取集合中元素的个数
    get()方法用于获取集合中某一个元素
    使用remove方法将下标为0的元素移除
    注意移除后，集合元素下标重写从0排列，即原来下标为1的元素下标变成了0
    <4> 里氏替换原则在上例中的应用
    add方法参数是Object类型
get方法返回值是Object类型
由于Object是任何类的父类，依据里氏替换原则，add方法可以添加任何类型的数据到集合中
此时被添加的数据会自动转换成Object类型
由于get方法返回的是Object类型，因此需要将从集合中获取的元素强制转换成原来的类型
    <5> 缺点
    类型的转换带来了效率的降低、也容易发生类型转换异常
  5. 泛型集合
    <1> 定义
    泛型是指类型参数化，即数据类型是可以变化的
    <2> e.g
    public static void main(String[] args) {
    // 定义泛型集合，该集合中只允许存储Person类型的对象
    List<Person> list = new ArrayList<Person>();
    // 向集合中添加不同类型的元素
    Person p1 =new Person();
    Person p2 =new Person();
    //向集合中添加元素
    list.add(p1);//add方法的参数是Person类型
    list.add(p2); //add方法的参数是Person类型
    // 显示集合中所有的元素
    for (int i = 0; i < list.size(); i++) {
        list.get(i).work();//get方法的返回值是Person类型
    }
}
注:
   list中只允许存储<>中规定的数据类型
  6. LinkedList类
    <1> 总
    和ArrayList的使用没有什么区别
    <2> 和ArrayList的区别
    a. ArrayList是实现了基于动态数组的数据结构，LinkedList基于链表的数据结构 
b. 对于随机访问get和set，ArrayList觉得优于LinkedList，因为LinkedList要移动指针 
c. 对于新增和删除操作add和remove，LinedList比较占优势，因为ArrayList要移动数据
    <3> 个人理解
    ArrayList相当于数组
LinkedList相当于链表
  7. Vector类
    <1> 总
    同ArrayList
    <2> 区别
    ArrayList和LinkedList是非线程安全的，并且是不同步的
Vector的所有方法都用了synchronized方法，是线程安全的
但是vector中的方法组合起来使用不是线程安全的
  8. Set接口
    <1> 和List接口的区别 
    List集合中存储的数据是可以重复的，有顺序的
Set集合中存储的数据是没有顺序的，不允许重复的
    <2> 常用方法
    add(Object obj)    加入元素
clear()    移除Set集合中所有元素
contains(Object obj)    判断Set集合中是否包含指定元素
isEmpty()    判断Set集合是否为空
iterator()    返回Set集合中对元素迭代的迭代器
remove(Object obj)    从集合中删除元素
size()    返回集合中的元素数量
    <3> e.g
    // 定义集合对象list
Set<Person> set = new HashSet<Person>();
//创建对象
Person p1 =new Person();
Person p2 =new Person();
Person p3 =new Person();
//向set集合中添加元素
set.add(p1);
set.add(p2);
set.add(p3);
set.add(p3);//p3在上一行已经添加到set中，再次添加会忽略。
  9. Iterator迭代器
    <1> 定义
    迭代也叫遍历，是指将集合或数组中所有的元素一一访问一遍
Set集合是没有下标的，因此不能使用for循环遍历
为此java提供了迭代器Iterator，专门用于集合的遍历
    <2> e.g
    Iterator<Person> iterator = set.iterator();
while(iterator.hasNext()){
  Person p = iterator.next();
  p.work();
}
    注:
    a. Iterator<Person> iterator = set.iterator()表示从set集合中获取迭代器
b. iterator.hasNext()表示是否还有未迭代完的元素，如果有返回true，否则返回false
c. iterator.next()表示从集合中迭代出一个集合元素
