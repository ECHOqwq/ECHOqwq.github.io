JSP基础知识整理1

发表于 2019-07-29 | 更新于 2019-08-09 | 分类于 Java | 评论数：
  1. JSP基础
    <1> JSP注释
    可以将要注释的内容用<%-- --%>括起来
拓:
    html注释 <!-- -->
    java注释 //         /* */
    <2> 和HTML注释的区别
    HTML注释会被JSP翻译引擎翻译到Servlet的out.write()中
JSP注释会被JSP翻译引擎忽略
    <3> 代码块
    声明的变量是不能添加访问权限的
不能定义方法
不能定义静态语句块
    <4> 声明语句块
    可以声明实例变量、实例方法、静态方法、静态代码、块等
    <5> 表达式块
    <%= %>
用来在JSP页面中输出变量、常量以及它们组成的各种表达式的值
因为是表达式不是语句，所以没有分号
其将被JSP引擎翻译到out.write()方法中输出
    <6> 总
    out.println()不能回车
要想回车，应用out.print("string"+"<br/>")
out.print()和<%= %>可以解析html代码
<%
        局部变量、java语句
%>
<%!
        全局变量、定义方法        
%>
    <7> e.g
    <%!
        public String name;
        public void init(){
                name = "fafa";
        }
%>
<%
        out.print("这本书的名字是"+name+"<br/>");
        <%=book%>
%>
  2. 九大内置对象
    <1> 定义
    在JSP的java代码块、表达式块等中可以直接使用的引用
注:
    为什么可以直接引用?
      JAVA代码块和表达式块被JSP引擎翻译后出现在Servlet的_jspService()方法中
      这九个对象是_jspService()方法中的局部变量
    <2> out对象(输出对象)
    out:输出流
out.print(x);
//输出x
    <3> request对象(请求对象)
    请求对象:存储"客户端向服务端发送的请求信息"
request对象的常见方法：
    String getParameter(String name) 根据请求的字段名key(input标签的name属性的value)，返回value属性值
    String[] getParameterValues(String name) 根据请求的字段名key，返回多个字段值
    void setCharacterEncoding("编码格式utf-8") 设置请求编码
    getRequestDispatcher().forward() 请求转发 A->B
    ServletContext getServerContext() 获取项目的ServletContext对象
    <4> response对象(响应对象)
    响应对象:存储"服务端向客户端响应的信息"
response对象的常见方法:
    void addCookie(Cookie cookie) 服务端向客户端增加cookie对象
    void sendRedirect(String location) throws IOException 页面跳转的方法(重定向)
    void sendContextType(String type) 设置服务端响应的编码(设置服务端的contentType类型)
    <5> 比较request和response里的页面跳转
    response.sendRedirect("a.jsp"); //页面跳转到a.jsp
如果要输出上一个页面的数据，数据会丢失，显示null，且网址会到/a.jsp
request.getRequestDispatcher("a.jsp").forward(request,response);//页面跳转到a.jsp
如果要输出上一个页面的数据，可以成功输出，网址不会到/a.jsp，会停留在上一个
    <6> Cookie(学session前需要了解) 
    客户端，不是内置对象
    Cookie对象是由服务端产生，再发送给客户端保存
    相当与本地缓存的作用，可以提高访问服务端的效率，但安全性较差
Cookie包含键值对: key=value
由javax.servlet.http.Cookie类产生
方法:
    public Cookie(String name,String value)
    String getName():获取name
    String getValue():获取value
    void setMaxAge(int expiry):最大有效期/s
服务端发送给客户端过程:
    服务端准备Cookie
            (放置)response.addCookie(Cookie cookie)
    页面跳转
    客户端获取Cookie
            request.getCookies();
注:
    服务端增加cookie:response对象
    客户端获取cookie:request对象
    不能直接获取某一个单独对象，只能一次性获取全部对象
通过f12可以发现，除了自己设置的Cookie对象外，还有一个name为JSESSIONID的cookie
    <7> session对象(会话对象)
    浏览网址：开始-关闭
购物：浏览-付款-退出
机制：
    客户端第一次请求服务端时，服务端会产生一个session对象(用于保存该客户信息)
    并且每个session对象都会有唯一一个sessionId(用于区分其他session)
    服务端又会产生一个cookie，并且该cookie的name=JSESSIONID，value=服务端sessionId
    然后服务端会在响应客户端的同时，把cookie发给客户端
    客户端的cookie就可以和服务端的session一一对应
    (JSESSIONID-sessionId)
注:
    session存储在服务端
    session是在同一个客户请求时共享
常用方法：
    String getId() 获取sessionId
    boolean isNew() 判断是不是第一次访问
    void invalidate() 使session失效
    void setAttribute() 赋值
    Object getAttribute() 取值
    void setMaxInactiveInterval 设置最大有效非活动时间/s
    int getMaxInactiveInterval() 获取最大有效非活动时间
若通过login.jsp利用session的setAttribute()赋了值，和login.jsp在同一个web项目的其他文件如a.jsp等都可以利用session.getAttribute()得到值
比如：我访问淘宝网站，第一次登陆以后，访问淘宝的任何子页面(比如某个产品页面)，我的登录信息就会一直存在
注意我如果用谷歌浏览器登陆了淘宝，我用IE浏览器打开淘宝是不会有登录记录的，但是如果我再开一个新的谷歌浏览器访问淘宝，这时候是存在登录记录的
    <8> application对象(全局对象)
    常用方法:
    String getContextPath() 当前项目的虚拟路径
    String getRealPath(String name) 当前项目的绝对路径
    <9> pageContext JSP页面容器
    <10> page 当前JSP页面对象(相当于java的this)
    <11> exception 异常对象
    <12> config 配置对象
    <13> 四个对象比较
    pageContext 当前页面有效
request     同一次请求有效
session     同一次会话有效
application 全局有效(整个项目有效)
共有方法:
        Object getAttribute(String name)
        void setAttribute(String name,Object obj)
        e.g
                setAttribute("a","b");
                如果a对象之前不存在，则新建一个a对象
                如果a存在，则将a的值改为b
        void removeAttribute(String name)
pageContext:仅在当前页面有效，页面只要跳转就无效
request:同一次请求有效，其他均无效(请求转发后有效，重定向后无效)
session:同一次会话有效(无论怎么跳转，都有效；关闭/切换浏览器后无效)
application:整个项目运行期间都有效(切换浏览器仍然有效，关闭服务器或重启其他项目后无效)
如果要进行多个项目共享或重启后仍然有效，可以通过JNDI实现
  3. JSP指令
    <1> 作用
    JSP指令为当前页面做一些基本的属性设置，提供基本环境
    <2> 分类
    page指令(页面指令)
include指令(包含指令)
taglib指令(标签库指令)
    <3> 统一语法格式
    <%@ 指令名称 属性名=属性值 属性名=属性值 …… %>
  4. page指令
    <1> 作用
    用于设置当前JSP页面的相关信息，一个JSP文件可以包含多个page指令
    <2> pageEncoding属性
    用于设置当前JSP页面所使用的的字符编码格式
即用户在浏览器中通过右击查看编码看到的格式
<%@ page pageEncoding = "utf-8" %>
注:
   pageEncoding属性和contentType属性的区别:
        <%@ page contentType = "text/html; charset = UTF-8" %>
        如果是pageEncoding默认使用的类型是text/html
        而contentType可以设置别的类型，比如text/xml
    <3> import属性
    用于在JSP页面中导入指定的类
<%@ page import = "java.util.Date" %>
即可在JSP中new一个Date()对象
    <4> errorPage属性
    <%@ page errorPage = "/error.jsp" %>
当当前页面出现错误时，自动跳转到另一个页面
该页面由error.jsp文件产生
注:
   关于/error.jsp前面的/问题(路径问题)
   路径分为两大类
     以斜杠开头的相对路径
     不以斜杠开头的相对路径
     其中以斜杠开头的相对路径又分为前台路径和后台路径
     出现在html或者jsp页面的静态路径称为前台路径
     出现在jsp的动态部分或者出现在java代码或者xml配置文件里的是后台路径
     后台路径的参照路径是当前web项目的根
     当前即是在webcontent文件夹下是否存在error.jsp文件
    <5> isErrorPage属性
    <%@ page isErrorPage = "true" %> (true和false值可改)
即当前页面是一个错误处理页面
错误处理页面里面会多出一个exception内置对象
exception对象里有一个getMessage()方法
可以通过使用该方法获取错误信息
ex = <%=exception.getMessage() %>
    <6> session属性
     使用request的getSession()的原则
        若向Session域中存放数据，则使用getSession(true)，即getSession()
        若从Session域中读取数据，则使用getSession(false)
而此处的session调的是request的无参getSession()
// session = pageContext.getSession();
e.g
//hello1.jsp
<%
        session.setAttribute("user","hahaha");
%>
//hello2.jsp
<%
        String user = (String)session.getAttribute("user");
        out.print("user ="+user);
%>
如果先访问hello1再访问hello2，out:user = hahaha
如果先访问hello2，out:user = null
    <7> contentType属性
    浏览器解析jsp的编码
  5. include指令
    <1> 作用
        用于将指定的文件包含到当前的JSP文件中
        该指令只有一个属性file，用于指定要包含的文件
        可以利用include的该作用做一个框架
<2> 用法
        被include指定包含的文件，可以是JSP动态页面文件
        也可以是HTML静态页面文件
<3> 运行过程
        使用include调用b文件的a文件先调用b文件过来再编译
        也叫静态联编
        静态联编后两个文件的变量是共享的
        也可以在b文件中使用a文件的变量
        但是注意使用的a文件的变量要在a文件的include指令后面出现
  6. JSP动作
    <1> 引入
        在JSP页面中大量使用java代码块、表达式等内容，会使JSP页面看起来很乱
        取而代之的是EL表达式、JSTL标签及JSP动作
<2> 定义
        使用系统定义好的标签完成本应由java代码来完成的功能
<3> 语法格式
        <jsp:动作名称 属性名=属性值 属性名=属性值 ……></jsp:动作名称>
        <jsp:动作名称 属性名=属性值 属性名=属性值 ……/>
<4> 常用动作
        转发动作&&包含动作

  7. forward动作

    <1> 简介
        页面一旦有了forward动作，当前页面所有要显示的内容都将无法显示
        因为页面直接转发到了下一页面，无法向标准输出流写入数据
        由目标页面给浏览器响应
<2> e.g
        //next.jsp
        next page<br>
        //index.jsp
        index page before<br>
        <jsp:forward page="/next.jsp"></jsp:forward>
        index page after<br>

        out: next page
        注:
           forward动作无法向标准输出流写入数据，所以只输出next.jsp中的东西

  8. include动作

    <1> 简介
        用于完成将指定页面包含到当前页面中的功能
        浏览器请求，include()动作发起页面，可以向标准输出流中写入数据
        并且由它自己响应浏览器，把目标页面包含到发起页面中
<2> e.g
        //next.jsp
        next page<br>
        //index.jsp
        index page before<br>
        <jsp:include page="/next.jsp"></jsp:include>
        index page after<br>

        out: index page before
             next page
             index page after
        注:
           include动作可以向标准输出流中写入数据
<3> 动态联编
        这是动态联编，变量之间不共享
        会形成两个servlet
<4> 静态联编和动态联编的应用
        一般使用静态联编，因为在程序运行时只存在一个servlet
        对资源的消耗较少，不存在调用问题，效率高
        若两个文件中需要共享同一个变量，使用静态联编
        如果两个文件中存在同名变量，使用动态联编
