AOP

发表于 2019-08-26 | 更新于 2019-08-27 | 分类于 Java | 评论数：
  1. AOP介绍
    Aspect Oriented Programming
    面向切面编程，通过预编译方式和运行期动态代理实现程序功能的统一维护的一种技术
    它是OOP（面向对象编程）的延续
    利用AOP可以对业务逻辑的各个部分进行隔离，使业务逻辑各部分之间耦合度降低
    AOP采取横向抽取机制，取代了传统纵向继承体系
    经典应用：性能监视、事务管理、安全检查、缓存
    Spring AOP使用纯Java实现，不需要专门的编译过程和类加载器
    在运行期通过代理方式向目标类植入增强代码
  2. AOP实现原理
    AOP底层将采用代理机制进行实现
    接口+实现类：spring采用jdk的动态代理Proxy
    实现类：spring采用cglib字节码增强
  3. AOP术语
    <1> target
    目标类，需要被代理的类
e.g: UserService
    <2> Joinpoint
    连接点，可能被拦截到的方法
e.g: 所有的方法
厕所里的坑（只知道有几个坑，但它们不一定被使用了）
    <3> PointCut
    切入点，已经被增强的连接点
e.g: addUser()
厕所里被使用的坑
    <4> advice
    通知/增强，增强代码
即拦截到Joinpoint之后要做的事情就是通知
e.g: after/before
    <5> Weaving
    织入，即把增强advic应用到目标对象target来创建新的代理对象proxy的过程
    <6> Proxy
    代理类
    <7> Aspect
    切面，是切入点和通知的结合
  4. JDK动态代理
    <1> JDK动态代理
    对“装饰者”设计模式简化
使用前提
    必须有接口
目标类：接口+实现类
切面类：用于存通知MyAspect
工厂类：编写工厂生成代理
  5. CGLIB字节码增强
    <1> 和动态代理的区别
    没有接口，只有实现类
采用字节码增强框架cglib(其中一种字节码增强框架)
在运行时创建目标类的子类，从而对目标类进行增强
    <2> 导入jar包
    需要把spring-core-（版本）.RELEASE.jar
复制到该项目的webroot目录下的web-inf的lib文件夹下
    <3> 注
    ctrl+f 替换/查找
  6. AOP联盟通知类型
    <1> 总
    AOP联盟通知为advice定义了org.aopalliance.aop.Advice
    <2> 前置通知
    org.springframework.aop.MethodBeforeAdvice
在目标方法执行前实施增强
    <2> 后置通知 
    在目标方法执行后实施增强
    <3> 环绕通知(最重要)
    org.aopalliance.intercept.MethodInterceptor
在目标方法执行前后实施增强
e.g 
    环绕通知必须手动执行目标方法
    try{
        //前置通知
        //执行目标方法
        //后置通知
    }catch(){
        //异常抛出通知
    }
    <4> 异常抛出通知
    在方法抛出异常后实施增强
    <5> 引介通知(了解)
    在目标类中添加一些新的方法和属性
  7. spring编写代理：半自动
    <1> 让spring创建代理对象，从spring容器中手动获取代理对象
    <2> 导入jar包
    核心：4+1
AOP：AOP联盟(规范)、spring-aop(实现)
    <3> 过程
    目标类
  8. spring aop编程：全自动
    <1> 从spring容器中获得目标类，如果配置aop，spring将自动生成代理
    <2> 要确定目标类，要通过aspectJ的切入点表达式
    <3> 导入jar包
    com.springsource.org.aspectj.weaver-1.6.8.RELEASE.jar
