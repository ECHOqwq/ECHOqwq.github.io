方法1：暴力法
//判断是否是回文序列
public static boolean isPalindromic(String s) {
int len = s.length();
for (int i = 0; i < len/2; i++) {
if (s.charAt(i) != s.charAt(len - i - 1)) {
return false;
}
}
return true;
}
public static void main(String[] args) {
Scanner sc = new Scanner(System.in);
String s = sc.next();
String ant = “”; //用于存储当前得到的最大的回文序列
String s2 = “”;//用于存储切出来的子序列
int max = 0;
for (int i = 0; i<s.length();i++) {
for (int j = i+1; j<=s.length();j++) {
s2 = s.substring(i,j);
if (isPalindromic(s2)&&s2.length()>max) {
ant = s2; //即为当前范围最大的回文序列
max = Math.max(max, ant.length());
}
}
}
System.out.println(ant);
}

方法2：动态规划
思想：前面计算出的子回文序列再次碰到时不用重新判断，第一次得到的时候就把它存储起来
思路：
利用了一个二维数组P[][]
若s[i]~s[j]是回文序列，则P[i][j]=true
若s[i]~s[j]不是回文序列，则P[i][j]=false
P[i][j]=(P[i+1][j-1]&&s[i]==s[j])
所以想要知道P[i][j]的状态，只需要知道P[i+1][j-1]，而不需要重新从中心看起
注意：
需要注意的是长度为1和长度为2的字符串，如果此时i+1,j-1，则会出现i>j的情况
所以len == 1 和 len == 2两个条件需要单独列出来

方法3：扩展中心
首先需要明确的是，字符串有2n-1个中心。
先来解释一下为什么中心是2n-1而不是n。
比如有字符串abcba，这时回文子串是abcba，中心是c；
又有字符串adccda，这时回文子串是adccda，中心是cc。
由此可见中心点既有可能是一个字符，也有可能是两个字符。
当中心为一个字符的时候有n个中心，当中心为两个字符的时候有n-1个中心，所以一共有n+n-1个中心。
public static void main(String[] args) {
Scanner sc = new Scanner(System.in);
String s = sc.next();
s = longestPalindrome(s);
System.out.println(s);
}
//得到最长的回文子串并返回
public String longestPalindrome(String s) {
if (s == null || s.length() < 1) return “”;
int start = 0, end = 0;
for (int i = 0; i < s.length(); i++) {
int len1 = expandAroundCenter(s, i, i);//一个字符为中心的情况
int len2 = expandAroundCenter(s, i, i + 1);//两个字符为中心的情况
int len = Math.max(len1, len2);//取最长的长度
if (len > end - start) {//根据当前i值和得到的len长度计算出当前的子字符串的开始和结束位置
start = i - (len - 1) / 2;
end = i + len / 2;
}
}
return s.substring(start, end + 1);
}
//返回当前得到的回文子串的长度
public static int expandAroundCenter(String s, int left, int right) {
int L = left, R = right;
while (L >= 0 && R < s.length() && s.charAt(L) == s.charAt(R)) {
L–;
R++;
}
return R - L - 1;
}
方法4：Manacher’s Algorithm 马拉车算法
首先我们解决下奇数和偶数的问题，在每个字符间插入 “#”，并且为了使得扩展的过程中，到边界后自动结束，在两端分别插入 “^” 和 “$”，两个不可能在字符串中出现的字符，这样中心扩展的时候，判断两端字符是否相等的时候，如果到了边界就一定会不相等，从而出了循环。经过处理，字符串的长度永远都是奇数了。
